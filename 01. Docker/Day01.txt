============
DOCKERFILE
============
Dockerfile has instructions to build the docker image for the custom applications
Dockerfile has DSL key-words
DSL - Domain Specific Language
All the instructions in the Dockerfile will be processed from top to bottom

Dockerfile has the below key-words;
FROM - It is used to setup the base image to run application
FROM ubuntu
FROM amazon-Linux
FROM python
FROM tomcat

RUN - It is used to execute the commands 'while creating the docker image'
RUN touch kastro.py
RUN mkdir kastro
RUN yum install git -y
RUN git clone <RepoURL>

CMD - It is used to execute the commands 'while creating the docker container'

Note: We can write multiple RUN instructions in the same Dockerfile and all the RUN instructions will be processed from TOP to BOTTOM
We can write multiple CMD instructions in the same Dockerfile, but only the last CMD instruction in the Dockerfile will be processed 
CMD touch kastro.py
CMD mkdir kastro
CMD yum install git -y
CMD git clone <RepoURL>

Ex:
RUN touch kastro.py
RUN mkdir kastro
CMD yum install git -y
RUN touch docker.java
CMD mkdir docker
RUN pwd

docker build -t image001 .
docker run -it --name container001 image001:latest

docker run -itd --name container001 image001:latest
docker exec -it afbe1bf49356 /bin/bash

COPY - it is used to copy the files/directories to a docker image while creating the image

ADD - it is also used to copy the files from source location to destination location to the image
ADD command can download the files from remote location (internet)

WORKDIR - it is used to set the working directory inside the container
Whatever commands that we write after WORKDIR, all those commands will get executed inside the Working Directory 

EXPOSE - it is used to expose our application for the public
EXPOSE 8080

ENTRYPOINT - All the entrypoint instructions will get executed when container starts
